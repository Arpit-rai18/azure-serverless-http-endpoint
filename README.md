# Serverless HTTP Endpoint with Azure Functions

## Overview

This project demonstrates how to create and deploy a serverless HTTP endpoint using Azure Functions. The function was configured with an HTTP trigger and deployed to Microsoft Azure. Application Insights was also enabled to monitor logs and function activity.

The project helped me understand serverless computing, Azure Function Apps, monitoring, and access restriction concepts in Azure.

---

## Technologies Used

* Microsoft Azure
* Azure Functions
* Azure Function App
* HTTP Trigger
* Application Insights
* Azure Cloud Shell

---

## Project Objectives

* Create and configure an Azure Function App
* Deploy an HTTP-triggered Azure Function
* Test the endpoint functionality
* Enable monitoring using Application Insights
* Review invocation logs
* Restrict access to the function endpoint

---

## Steps Performed

### 1. Prepared the Azure Environment

* Signed in to the Azure Portal
* Opened Azure Cloud Shell
* Configured the required Azure resources

### 2. Created the Function App

* Created a new Azure Function App
* Configured runtime settings and hosting options

### 3. Developed the HTTP Trigger Function

* Created a serverless HTTP-trigger function
* Configured request handling and response output

### 4. Deployed the Function to Azure

* Published the function project to Azure
* Verified successful deployment

### 5. Tested the HTTP Endpoint

* Accessed the endpoint through the browser
* Validated successful execution of the function

### 6. Enabled Monitoring and Logs

* Enabled Application Insights
* Reviewed invocation logs and monitoring data

### 7. Restricted Access

* Configured access restrictions for the function
* Tested restricted access behavior
